# 书脊数据标注规范

## 1. 标注目标

每个标注实例代表图像中一本真实、独立的图书。使用多边形勾勒可见书脊区域，
最终实例数量同时作为该图像的图书数量真值。

类别固定为：

```text
book_spine
```

不得按书名建立多个检测类别。书名属于 OCR 和馆藏匹配属性，不是 YOLO 类别。

## 2. 多边形绘制规则

- 沿书脊实际可见边缘绘制多边形，尽量贴合边界。
- 正常矩形书脊通常使用 4 个点；存在透视或弯曲时可增加必要点位。
- 不要把相邻两本书合并成一个多边形。
- 同名书并排摆放时，每一本都必须独立标注。
- 书脊被标签贴纸遮挡但整体轮廓清楚时，仍标注完整可见轮廓。
- 书脊部分超出画面时，只标注画面内的可见部分。
- 书脊与背景边界无法判断，或可见面积不足约 30% 时，不做正式标注，
  并在人工检查备注中记录。
- 横放书籍若能识别为独立书脊，同样标注；不要只标竖直书籍。
- 书架隔板、装饰盒、空隙、书封正面和非图书物体不得标注为书脊。

## 3. 书名处理规则

当前阶段 Label Studio 只标注书脊多边形，不再人工填写书名。

原因是 YOLOv8n-seg 训练只需要 `book_spine` 的实例分割标签；书名识别属于后续
OCR、文本清洗和馆藏目录匹配模块。这样可以先快速完成 200 张书脊分割数据集，
避免把每本书名都手工录入一遍。

后续书名处理路线为：

```text
书脊分割 → 书脊裁剪/旋转校正 → PaddleOCR → 馆藏目录相似度匹配 → 人工确认低置信度结果
```

## 4. 数量标注

每张图片的数量真值等于该图中有效 `book_spine` 多边形数量，因此无需再手工
填写一个容易与实例数不一致的总数。导出标注后由转换脚本自动生成：

```text
file_name,book_count
IMG_7832.jpg,18
```

跨图像重复出现的同一本书，在各自图片中都要正常标注。跨图去重属于盘点算法
任务，不能通过漏标来“提前去重”。

## 5. 质量复核

每张图片至少完成一次自检，建议由另一名组员进行抽查：

- 是否存在漏标、重复标注或把两本书合并的情况；
- 多边形是否越过相邻书脊或包含大面积背景；
- 书名是否使用规范题名；
- 同名复本是否分别建立实例；
- 横放、倾斜、边缘截断和遮挡书籍是否按统一规则处理。

建议至少随机抽查 20% 图像。发现规则理解不一致时，应先统一标注规范，再继续
大批量标注，避免后期返工。

## 6. 场景编号和数据划分

- 连续拍摄同一层书架的一组图片使用同一个 `scene_id`。
- 更换书架、楼层、拍摄方向或明显不同区域时创建新的场景编号。
- 推荐格式：`S001`、`S002`、`S003`。
- 训练集、验证集和测试集必须按完整场景划分。
- 禁止把同一书架的相邻重叠图片随机分入不同集合，否则会造成数据泄漏和虚高指标。

## 7. Label Studio 使用

1. 使用 [book_spine.xml](../configs/label_studio/book_spine.xml) 配置标注界面。
2. 运行 `scripts/prepare_label_studio_tasks.py` 生成任务 JSON。
3. 导入任务后，按文件名顺序进行标注。
4. 完成阶段性标注后导出 Label Studio JSON，并保留原始导出文件。
5. 后续使用专门脚本转换为 YOLO 分割格式，不手工编辑 YOLO 标签。

当前项目已提供启动脚本：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start_label_studio.ps1
```

启动后访问：

```text
http://127.0.0.1:8080
```

导入任务文件：

```text
data/interim/label_studio_tasks.json
```

若图片无法显示，优先检查启动服务时是否使用了本项目脚本。该脚本会自动开启
Label Studio 的本地文件访问，并把本项目根目录设置为本地文件根目录。

## 8. 模型预标注修正规则

当前项目曾使用 65 张人工标注训练 v2 辅助模型并尝试生成预标注，但实际检查发现
效果较差，修改成本高于重画，因此已清除这批预标注。后续如再次尝试预标注，应先
小批量验证，不要直接写入全部未完成任务。

- 预标注只是草稿，不等于正式标注；必须人工检查后点击 Submit。
- 如果模型把书架隔板、空隙、书封正面或背景误标为书脊，直接删除该区域。
- 如果模型漏掉某本书，手工补一个 `book_spine` 多边形。
- 如果模型边界明显压到相邻书脊或包含大面积背景，拖动点位微调。
- 如果边缘只露出一小部分但可见面积约超过 30%，按可见部分保留；过小或无法判断则删除。
- 同名书并排仍然保留多个独立书脊区域，不要合并。

小批量预标注测试命令示例：

```powershell
python scripts/generate_label_studio_predictions.py --conf 0.45 --polygon-mode min-rect --clear-existing
```

该命令默认使用：

```text
models/weights/book_spine_v2_65_best.pt
```

其中 `--polygon-mode min-rect` 会把模型 mask 转成 4 点旋转矩形，比原始分割轮廓
更容易在 Label Studio 中拖动修改。若试标 5–10 张后发现仍然需要大面积删除和重画，
应放弃预标注，继续人工标注，避免把时间花在修劣质结果上。
