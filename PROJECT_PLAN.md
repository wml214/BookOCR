# 基于视觉的图书盘点系统实施计划

## 1. 项目目标

构建一套面向西院图书馆书架场景的多图图书盘点系统。系统接收按拍摄顺序上传的多张图像，完成书脊分割、OCR 识别、馆藏目录纠错、相邻图像去重和数量汇总，并通过 Streamlit 界面展示及导出结果。

本项目必须满足实验指导书中的硬性要求：

- 图像数据集不少于 200 张。
- 完成书脊轮廓、书名和数量标注。
- 至少训练一个神经网络模型。
- 软件具有完整、合理且可演示的可视化功能。
- 提交标注数据、完整工程代码、训练模型、模型参数和课程论文。
- 课程论文严格使用《电子学报》模板。

## 2. 技术路线

### 2.1 数据处理

1. 将现有 HEIC 原图批量转换为高质量 JPEG，保留原图作为只读来源。
2. 补采至少 118 张，建议最终保留 220 至 250 张有效图像。
3. 拍摄时按书架从左到右、从上到下排列，相邻图像保留 20% 至 30% 重叠区域。
4. 使用 Label Studio 或等价工具标注书脊多边形，并记录规范书名。
5. 按书架场景划分训练集、验证集和测试集，比例为 70%/15%/15%；同一连续书架不得跨数据集。

### 2.2 书脊分割

- 初始模型采用 `YOLOv8n-seg`。
- 初始训练分辨率为 640，启用 AMP 混合精度，使用 RTX 4060 8GB GPU。
- 优先使用自动 batch；如显存不稳定，从 batch 8 开始逐步下调。
- 记录 Precision、Recall、mAP@0.5、mAP@0.5:0.95、训练曲线和误差案例。
- 若小模型精度不足，再对比 `YOLOv8s-seg` 或提高输入分辨率，不直接使用大模型。

### 2.3 OCR 与馆藏匹配

1. 根据分割多边形裁剪书脊，并进行透视矫正、方向校正和对比度增强。
2. 使用 PaddleOCR 识别中文、英文和数字文本。
3. 从馆藏 Excel 中提取题名、责任者、索书号、出版社等必要字段，生成精简的 `catalog.csv`。
4. 规范化 OCR 文本：统一大小写、简繁体、空白和标点。
5. 使用编辑距离、字符相似度和关键字覆盖率对候选书名排序。
6. 低于匹配阈值的结果标为“待确认”，由界面提供人工修正入口。

### 2.4 多图去重与计数

1. 对相邻图像提取 ORB 特征并估计单应性矩阵。
2. 将相邻图像中的书脊区域映射到公共坐标系。
3. 结合空间重合度、书名相似度和拍摄顺序判断是否为同一实体书。
4. 同名但空间位置不同的书必须分别计数。
5. 无法可靠配准时不强行合并，并在结果中输出重复计数风险提示。

### 2.5 可视化软件

Streamlit 软件应包含：

- 多图上传、排序和删除。
- 模型和馆藏目录状态检查。
- 一键执行书脊分割、OCR、目录匹配和去重计数。
- 原图与标注图对照查看。
- 识别结果、置信度和来源图像展示。
- 待确认书名的人工修正。
- 按规范书名汇总册数。
- CSV 导出和运行日志展示。

## 3. 工程阶段

### 阶段 A：工程骨架与数据准备

- 建立统一目录、配置、日志和输出规范。
- 将馆藏 Excel 转换为去重后的精简 CSV。
- 将 HEIC 转换为 JPEG，并生成图像清单。
- 制定标注规范和场景编号规则。

验收标准：预处理脚本可重复执行，原始文件不被修改，数据统计可追溯。

### 阶段 B：标注与数据集构建

- 完成不少于 200 张有效图像。
- 完成书脊多边形、书名和数量标注。
- 将标注转换为 YOLO 分割格式。
- 按场景完成训练、验证和测试划分。

验收标准：随机抽查标注无明显漏标、错标或越界，多边形和图像一一对应。

### 阶段 C：模型训练与评估

- 完成 YOLOv8n-seg 基线训练。
- 调整图像尺寸、增强策略和训练轮数。
- 保存最佳权重、配置、训练日志和指标图。

目标指标：书脊分割 mAP@0.5 不低于 85%。

### 阶段 D：OCR、目录匹配与去重

- 完成书脊裁剪和方向处理。
- 完成 PaddleOCR 封装。
- 完成馆藏候选检索与置信度计算。
- 完成相邻图像配准和实体书去重。

目标指标：书名匹配准确率不低于 90%，最终计数准确率不低于 90%。

### 阶段 E：软件集成

- 接通完整推理流水线。
- 实现结果查看、人工纠错、汇总和 CSV 导出。
- 加入输入检查、异常处理和运行日志。

验收标准：从多图上传到盘点结果导出的完整流程可稳定演示。

### 阶段 F：论文与答辩

- 整理数据、模型、OCR、匹配、计数和软件实验结果。
- 按《电子学报》模板撰写摘要、引言、方法、实验和结论。
- 准备 5 至 8 分钟演示和常见问题回答。
- 按实验指导书要求打包代码、权重和软件，不把原始图片放入代码压缩包。

## 4. 测试计划

- 清晰单图：检测、识别和计数结果正确。
- 竖排、横排、倾斜和低对比度书脊：方向校正有效。
- 相邻重叠图像：同一本实体书只计一次。
- 两本同名书并排：应计为两册。
- 馆藏目录中不存在的书：标为未知或待确认。
- 空书架、模糊图像和非书籍图像：软件不崩溃并给出明确提示。
- 馆藏 Excel 缺列、图像损坏或模型文件缺失：输出可理解的错误信息。
- 独立测试集：计算分割、OCR、书名匹配、计数和推理耗时指标。

## 5. 代码质量约定

- 所有核心模块、复杂算法、输入输出和异常处理必须有详细中文注释。
- 函数和类使用类型标注，并为公共接口编写中文 docstring。
- 路径、阈值和模型参数统一由配置文件管理，不在业务代码中散落魔法数字。
- 每个阶段都保留可重复执行的脚本和日志。
- 原始数据只读，所有转换结果写入 `data/interim` 或 `data/processed`。
- 单元测试放在 `tests`，测试数据必须小且可公开，不依赖完整原始数据。

- 开发环境、工程骨架和界面原型：4 课时。
